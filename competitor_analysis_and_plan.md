## 2. Our Strategic Differentiator: The "Sandbox-First" Auditor

Instead of just outputting text summaries (like CodeRabbit or Greptile), Astra Vision's unique advantage is the **Interactive Line Sandbox**. A developer can click on *any* flagged issue in a PR review and immediately run that exact snippet with mock inputs in the browser. 

---

## 3. Technical Feature Roadmap (Proposals)

Here is a list of features we need to build next to bridge the gap and surpass our competitors:

### Proposal A: Inline Monaco Diagnostics (The "IDE Feel")
* **What**: Display Nemotron code review warnings directly inside our Monaco Editor instance using red/yellow squiggly lines (`markers` API). Hovering over a line displays the warning and a button: `[Simulate in Sandbox]`.
* **Why**: Avoids the "side panel fatigue" that developers experience with traditional review UIs.
* **Tech**: Monaco Editor `monaco.editor.setModelMarkers` API.

### Proposal B: Autonomous Self-Healing Sandbox (The "Run & Fix" Loop)
* **What**: When a user clicks `Run Audit` and a `Critical` or `Warning` bug is found, the backend will automatically generate a mock JS/Python unit test, run it inside the sandbox, catch the failure, and have Gemini write the corrected fix.
* **Why**: Replicates Sweep's "self-healing code" capability directly within our Web UI, so developers can see the bug fail and pass dynamically.
* **Tech**: Backend Pyodide/Node execution runner + test assertions generated by Gemini.

### Proposal C: AST Dependency Graph (The "Greptile Killer")
* **What**: Upgrade our `code_indexer.py` from basic text chunking to building an Abstract Syntax Tree (AST) dependency graph.
* **Why**: Allows Astra Vision to trace exact import paths (e.g., *Function X in helper.js is imported on line 12 of index.js*), boosting our multi-file bug detection accuracy to 85%+.
* **Tech**: `esprima` (JS) or `jedi` (Python) parser on the backend.
